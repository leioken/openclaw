# 🛡️ 安全检查清单

灵感来自 evilcos 的安全研究思维

## 每次外部操作前检查

### 🔐 认证与授权
- [ ] API Key 是否正确配置且未过期
- [ ] 用户是否已授权此操作
- [ ] 是否有最小权限原则

### 🔒 数据安全
- [ ] 是否涉及敏感数据（密码、token、个人信息）
- [ ] 数据是否加密传输
- [ ] 是否避免了日志泄露敏感信息

### 📦 第三方依赖
- [ ] 是否审查了第三方代码/技能
- [ ] 是否有已知安全漏洞
- [ ] 是否来自可信来源

### 🌐 网络操作
- [ ] URL 是否可信（避免 SSRF）
- [ ] 是否验证了 SSL 证书
- [ ] 是否有超时和重试限制

### 💾 文件操作
- [ ] 文件路径是否安全（避免路径遍历）
- [ ] 是否有写入权限检查
- [ ] 是否使用了回收站而非直接删除

## 常见攻击向量防护

### XSS/注入攻击
- 不执行用户提供的任意代码
- 验证所有输入参数
- 使用参数化查询

### CSRF/未授权访问
- 验证操作来源
- 使用 token 验证
- 检查用户权限

### 信息泄露
- 不在日志中打印敏感数据
- 不在错误消息中暴露内部信息
- 限制错误详情返回给用户

### 资源耗尽
- 设置合理的超时时间
- 限制请求频率
- 限制文件大小和处理时间

## 响应流程

发现安全问题时：
1. **立即停止** 可疑操作
2. **记录详情** 时间、操作、影响范围
3. **通知用户** 清晰说明问题和风险
4. **等待指示** 不擅自继续风险操作

## 安全格言

> "Security is a process, not a product." - 安全是过程，不是产品
