## 调用方式

```bash
# 手动指定代理
/model bailian/qwen3-coder-plus

# 或通过会话标签
sessions_spawn --label "code-review" --task "审查这个 PR"
```

## 团队协作流程

1. **接收任务** → 主代理分析
2. **任务分类** → 判断是否需要专业代理
3. **分配执行** → 调用对应代理
4. **结果整合** → 主代理统一回复

## 示例场景

### 场景 1: 代码审查
```
用户：帮我审查这个 PR
→ 主代理调用代码专家
→ 代码专家分析代码
→ 主代理整合建议并回复
```

### 场景 2: 日常对话
```
用户：今天天气如何？
→ 主代理直接处理
→ 调用天气技能
→ 回复用户
```

### 场景 3: 复杂项目
```
用户：帮我建一个待办应用
→ 主代理调用代码专家
→ 代码专家编写代码
→ 主代理解释设计思路
→ 持续迭代开发
```

---

**原则:** 主代理负责协调和沟通，专业代理负责特定领域的深度工作。
